## ✨ Highlights

- **Scenarios**: Sign‑Up, Sign‑In, Bank Features (Transfer Funds, Bill Pay, Request Loan, Account Overview, Logout)
- **Workload Models**: 
  - *Sign‑In – Concurrency Thread Group* (constant concurrent users)
  - *Sign‑Up – Ultimate Thread Group* (phased ramps; Groovy data generation)
  - *Basic Thread Groups* for fast functional/perf smoke
- **Controllers**: Transaction Controller (timing groups), **Modular Controller** for reusable “Bank Features”, Recording Controller, Interleave Controller demo
- **Data Strategy**:
  - Groovy generators for synthetic user data (sign‑up)
  - **CSV DataSet Config** for Sign‑In (credentials rotation, EOF behaviour)
- **Quality Gates**: Assertions at sampler + flow level (status/latency/body), optional SLA thresholds for p95/p99
- **Reporting**: JMeter HTML Dashboard per run, **Jenkins Performance Plugin** trend graphs per job
- **CI Orchestration**: Headless non‑GUI runs, clean workspace, archived reports and JTLs
- **Extensible**: Drop‑in JMX modularity + environment‑driven properties

## 🚀 How to Run Locally (non‑GUI)

Prereqs: Java 8+; JMeter 5.6.3 in `D:\apache-jmeter-5.6.3` (or export `JMETER_HOME`).

```bat
REM Windows (CMD/PowerShell)
set JMETER_HOME=D:pache-jmeter-5.6.3
set JMX=JMX files\Test1.jmx
set OUT=results\Test1.jtl
set RPT=html report\Test1

"%JMETER_HOME%in\jmeter.bat" -n -t "%JMX%" -l "%OUT%"
"%JMETER_HOME%in\jmeter.bat" -g "%OUT%" -o "%RPT%"
```

Tips:
- Use `-Jusers=50 -Jramp=60` style **user properties** in JMX (and bind to Thread Groups) to parametrize load.
- Clear old results (`/results`, `/html report`) to avoid mix‑ups between runs.

## 🤖 Jenkins CI

**Key environment variables** (example):
```groovy
environment {
  JMETER_HOME    = 'D:\apache-jmeter-5.6.3'
  JMX_DIR        = 'D:\jmeter project\JMX files'
  WS_RESULTS_DIR = 'jmeter-results'    // copied JTLs for Performance plugin
  WS_REPORT_DIR  = 'jmeter-html-report'// archived HTML dashboard
  JMETER_ADDOPTS = '-Djava.awt.headless=true'
}
```

**Typical stages**:
1. **Prep** – verify JMeter, clean workspace folders, mkdir results/report dirs
2. **Run** – non‑GUI `jmeter.bat -n -t ... -l ...` (optionally **per‑scenario** matrix)
3. **HTML Dashboard** – `-g JTL -o reportDir`, then copy to `WS_REPORT_DIR`
4. **Publish** – Jenkins **HTML Publisher** -> “JMeter HTML Dashboard”
5. **Trend** – Jenkins **Performance Plugin** picks up `WS_RESULTS_DIR/*.jtl` for trends
6. **Gates (optional)** – mark build **UNSTABLE/FAIL** if p95 regression exceeds thresholds

**Performance Plugin pattern**: `jmeter-results/*.jtl`  
**HTML Publisher dir**: `jmeter-html-report`

> Stuck builds? Use stage‑level `timeout()`; prefer `returnStatus` in scripted steps to avoid hard fails while still gating on metrics.

## 📊 Metrics, SLAs & Assertions

- **Latency & Throughput**: Transactions grouped via **Transaction Controller** for business‑level timing.
- **Assertions**:
  - Response Code = 200/302 as expected
  - Response Time `<` SLA (e.g., 95th percentile < **1.5s** for Sign‑In, < **2.0s** for Bank flows)
  - Body text contains key markers (e.g., “Accounts Overview”, “Bill Payment Complete”)
- **Suggested Gates** (example envs):
  - `P95_UNSTABLE_PCT=5` (mark Unstable if p95 grows >5% vs previous build)
  - `P95_FAIL_PCT=10` (mark Failed if p95 grows >10%)

## 🧪 Data Strategy

- **Groovy** (Sign‑Up): unique usernames/emails per sampler using time + counter patterns; random but reproducible if seeded.
- **CSV Data Set** (Sign‑In): rotate credentials; `Recycle on EOF = true/false` based on test type; `Stop thread on EOF` for strict capacity tests.

Example Groovy snippet (conceptual):
```groovy
def uid = (vars.get('RUN_ID') ?: System.currentTimeMillis().toString()) + '_' + vars.get('__threadNum')
vars.put('username', 'user_' + uid)
vars.put('email', uid + '@example.test')
```

## ➕ Extending the Suite

- Add a new **Thread Group** per business scenario and reference **Modular Controller** blocks for reuse.
- Parameterize user load with `-J` properties; wire them to Thread Group fields.
- Keep GUI listeners **disabled** while running in CI (HTML dashboard is generated by CLI).
- For DB flows, use **JDBC Sampler** + proper timeouts; store credentials in Jenkins credentials store.

## 🛠 Troubleshooting

- **“was unexpected at this time.”** → Quote paths with spaces; prefer short, space‑free dirs.
- **JMeter not found** → Validate `JMETER_HOME` and that `bin\jmeter.bat` exists.
- **HTML report empty** → Ensure JTL is non‑zero and not corrupted; generate report in a **fresh** folder.
- **Jenkins trend missing** → Confirm `jmeter-results/*.jtl` exists **in workspace**, not only external drive.
- **Run hangs** → Add per‑stage `timeout()`, set connect/read **timeouts** on HTTP/JDBC samplers, use **Stop Test Now** only as last resort.
