## üöÄ How to Run Locally (non‚ÄëGUI)

Prereqs: Java 8+; JMeter 5.6.3 in `D:\apache-jmeter-5.6.3` (or export `JMETER_HOME`).

```bat
REM Windows (CMD/PowerShell)
set JMETER_HOME=D:pache-jmeter-5.6.3
set JMX=JMX files\Test1.jmx
set OUT=results\Test1.jtl
set RPT=html report\Test1

"%JMETER_HOME%in\jmeter.bat" -n -t "%JMX%" -l "%OUT%"
"%JMETER_HOME%in\jmeter.bat" -g "%OUT%" -o "%RPT%"
```

Tips:
- Use `-Jusers=50 -Jramp=60` style **user properties** in JMX (and bind to Thread Groups) to parametrize load.
- Clear old results (`/results`, `/html report`) to avoid mix‚Äëups between runs.

## ü§ñ Jenkins CI

**Key environment variables** (example):
```groovy
environment {
  JMETER_HOME    = 'D:\apache-jmeter-5.6.3'
  JMX_DIR        = 'D:\jmeter project\JMX files'
  WS_RESULTS_DIR = 'jmeter-results'    // copied JTLs for Performance plugin
  WS_REPORT_DIR  = 'jmeter-html-report'// archived HTML dashboard
  JMETER_ADDOPTS = '-Djava.awt.headless=true'
}
```

**Typical stages**:
1. **Prep** ‚Äì verify JMeter, clean workspace folders, mkdir results/report dirs
2. **Run** ‚Äì non‚ÄëGUI `jmeter.bat -n -t ... -l ...` (optionally **per‚Äëscenario** matrix)
3. **HTML Dashboard** ‚Äì `-g JTL -o reportDir`, then copy to `WS_REPORT_DIR`
4. **Publish** ‚Äì Jenkins **HTML Publisher** -> ‚ÄúJMeter HTML Dashboard‚Äù
5. **Trend** ‚Äì Jenkins **Performance Plugin** picks up `WS_RESULTS_DIR/*.jtl` for trends
6. **Gates (optional)** ‚Äì mark build **UNSTABLE/FAIL** if p95 regression exceeds thresholds

**Performance Plugin pattern**: `jmeter-results/*.jtl`  
**HTML Publisher dir**: `jmeter-html-report`

> Stuck builds? Use stage‚Äëlevel `timeout()`; prefer `returnStatus` in scripted steps to avoid hard fails while still gating on metrics.

## üìä Metrics, SLAs & Assertions

- **Latency & Throughput**: Transactions grouped via **Transaction Controller** for business‚Äëlevel timing.
- **Assertions**:
  - Response Code = 200/302 as expected
  - Response Time `<` SLA (e.g., 95th percentile < **1.5s** for Sign‚ÄëIn, < **2.0s** for Bank flows)
  - Body text contains key markers (e.g., ‚ÄúAccounts Overview‚Äù, ‚ÄúBill Payment Complete‚Äù)
- **Suggested Gates** (example envs):
  - `P95_UNSTABLE_PCT=5` (mark Unstable if p95 grows >5% vs previous build)
  - `P95_FAIL_PCT=10` (mark Failed if p95 grows >10%)

## üß™ Data Strategy

- **Groovy** (Sign‚ÄëUp): unique usernames/emails per sampler using time + counter patterns; random but reproducible if seeded.
- **CSV Data Set** (Sign‚ÄëIn): rotate credentials; `Recycle on EOF = true/false` based on test type; `Stop thread on EOF` for strict capacity tests.

## ‚ûï Extending the Suite

- Add a new **Thread Group** per business scenario and reference **Modular Controller** blocks for reuse.
- Parameterize user load with `-J` properties; wire them to Thread Group fields.
- Keep GUI listeners **disabled** while running in CI (HTML dashboard is generated by CLI).
- For DB flows, use **JDBC Sampler** + proper timeouts; store credentials in Jenkins credentials store.

## üõÝ Troubleshooting

- **‚Äúwas unexpected at this time.‚Äù** ‚Üí Quote paths with spaces; prefer short, space‚Äëfree dirs.
- **JMeter not found** ‚Üí Validate `JMETER_HOME` and that `bin\jmeter.bat` exists.
- **HTML report empty** ‚Üí Ensure JTL is non‚Äëzero and not corrupted; generate report in a **fresh** folder.
- **Jenkins trend missing** ‚Üí Confirm `jmeter-results/*.jtl` exists **in workspace**, not only external drive.
- **Run hangs** ‚Üí Add per‚Äëstage `timeout()`, set connect/read **timeouts** on HTTP/JDBC samplers, use **Stop Test Now** only as last resort.
